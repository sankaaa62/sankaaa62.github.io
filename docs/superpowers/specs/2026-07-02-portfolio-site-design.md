достижения (из CV/LinkedIn), стек, ссылки на сторы/Steam.

## Контент и медиа

- Тексты о проектах составляются из CV (`G:\Саша\Kandakov_CV(en|ru).pdf`) и LinkedIn, на двух языках.
- Медиа архивных проектов копируются (только копирование, оригиналы не трогать)
  из `D:\DevArchive\BlackGames\MyProjects(BlackGames)` — иконки, скриншоты, видео.
- Видео сжимаются в webm 2–4 МБ (ffmpeg), чтобы репозиторий остался < 1 ГБ
  и файлы < 100 МБ (лимиты GitHub).
- CV-файлы кладутся в сайт для скачивания.

## Архитектура

- Astro c content collections: каждый проект = markdown-файл с frontmatter
  (название, роль, годы, стек, метрики, ссылки, медиа) в двух локалях.
- Компоненты: Hero, FeaturedProjectCard, PrototypeGrid, SkillTags, Footer,
  LanguageSwitcher.
- Никакого бэкенда, форм и БД. Полностью статично.
- Деплой: GitHub Actions → GitHub Pages при push в main.

## Обработка ошибок / ограничения

- Если для прототипа нет видео — карточка со статичной иконкой/скриншотом.
- Внешние видео (YouTube) — с постер-кадром, чтобы страница работала и без них.
- Сайт должен корректно работать на мобильных (рекрутеры открывают с телефона).

## Тестирование

- Локальный предпросмотр (`astro dev`) и визуальная проверка обеих локалей.
- Проверка сборки в CI перед деплоем.
- Lighthouse-проверка производительности (тяжёлые медиа — главный риск).
